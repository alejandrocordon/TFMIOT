Training Pipeline

MLForge soporta entrenamiento con PyTorch y TensorFlow/Keras, con mixed precision automatico, callbacks y tracking de metricas.

Flujo de Entrenamiento

Flujo completo del entrenamiento: desde config hasta checkpoints y metricas

Fuentes de Datos

📦

torchvision

Datasets integrados: flowers102, cifar10, cifar100, food101, stanford_cars. Descarga automatica.

data:
  source: torchvision
  dataset: flowers102
🤗

HuggingFace

Cualquier dataset de imagen del Hub: beans, food101, username/my_dataset.

data:
  source: huggingface
  dataset: beans
📁

Local (ImageFolder)

Directorio local con una carpeta por clase. Ideal para datasets propios.

data:
  source: local
  data_dir: ./mi_dataset/

Data Augmentation

Mixed Precision Training (AMP)

MLForge activa automaticamente mixed precision cuando hay GPU CUDA disponible. Esto reduce el uso de memoria y acelera el entrenamiento.

Configuracion Completa de Training

training:
  epochs: 20                  # Numero de epochs
  batch_size: 32              # Tamano del batch
  optimizer: adam              # adam | adamw | sgd
  learning_rate: 0.001        # Learning rate inicial
  scheduler: cosine           # cosine | step | plateau | none
  early_stopping:
    patience: 5                # Epochs sin mejora antes de parar
    metric: val_accuracy      # Metrica a monitorear
  output_dir: ./outputs       # Donde guardar checkpoints
  device: auto                # auto | cuda | cpu | mps
