Quickstart: Tu Primer Modelo en 5 Minutos

Esta guia te lleva desde la instalacion hasta tener un modelo entrenado y listo para desplegar.

1. Instalacion

# Clonar el repositorio
git clone https://github.com/alejandrocordon/tfmiot.git
cd tfmiot/mlforge

# Instalar con todas las dependencias
pip install -e ".[all]"

# Verificar instalacion
mlforge info
Dependencias opcionales: Instala solo lo que necesites con pip install -e ".[tensorflow]", ".[export]", o ".[dashboard]".

2. Crear un Proyecto

# Crear proyecto de clasificacion de imagenes
mlforge new mi_clasificador --task classification

# Se genera esta estructura:
mi_clasificador/
  ├── config.yaml          # Configuracion completa del pipeline
  ├── train.py             # Script de entrenamiento standalone
  ├── README.md
  ├── data/                # Datos (se descargan automaticamente)
  ├── outputs/             # Checkpoints y metricas
  └── exported_models/     # Modelos exportados

3. Configurar (config.yaml)

El archivo config.yaml controla todo el pipeline:

project:
  name: flower_classifier
  task: classification

data:
  source: torchvision          # torchvision | huggingface | local
  dataset: flowers102         # Dataset a usar
  input_size: 224
  augmentation:
    - horizontal_flip
    - random_rotation: 15
    - color_jitter: {brightness: 0.2}

model:
  architecture: mobilenet_v3_small  # Ver todas con: mlforge info
  pretrained: true              # Transfer learning
  num_classes: 102
  framework: pytorch            # pytorch | tensorflow

training:
  epochs: 20
  batch_size: 32
  optimizer: adam               # adam | adamw | sgd
  learning_rate: 0.001
  scheduler: cosine             # cosine | step | plateau
  early_stopping:
    patience: 5

export:
  formats:
    - onnx: {}
    - tflite: {quantize: int8}
    - coreml: {}

4. Entrenar

mlforge train --config config.yaml
Salida del entrenamiento con metricas en tiempo real

5. Exportar

# Exportar a todos los formatos
mlforge export --config config.yaml --formats all

# Benchmark comparativo
mlforge benchmark --model-dir exported_models/ --config config.yaml
Un modelo PyTorch se exporta a 5 formatos de despliegue optimizados

6. Desplegar

# Generar app Android
mlforge deploy android --model model.tflite --labels "rosa,tulipan,girasol"

# Generar app Web
mlforge deploy web --model model.onnx --labels "rosa,tulipan,girasol"

# O lanzar un servidor de inferencia
mlforge serve --model model.onnx --port 8080
Listo! Tienes un modelo entrenado, exportado y desplegado. Consulta las secciones detalladas para personalizar cada paso.
