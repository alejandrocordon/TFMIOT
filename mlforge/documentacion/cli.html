CLI Reference

MLForge proporciona 9 comandos para controlar todo el pipeline de ML desde la terminal.

mlforge new CORE

Crea un nuevo proyecto MLForge con estructura de directorios, config.yaml y scripts.

mlforge new <nombre> [opciones]

# Opciones:
  --task, -t       Tipo de tarea: classification, detection  [default: classification]
  --framework, -f  Framework: pytorch, tensorflow              [default: pytorch]
  --output, -o     Directorio padre                             [default: .]

# Ejemplos:
mlforge new flower_detector --task classification
mlforge new road_signs --task detection --framework tensorflow

mlforge train CORE

Entrena un modelo segun la configuracion YAML. Detecta automaticamente si usar PyTorch o TensorFlow.

mlforge train --config <path>

# Opciones:
  --config, -c     Ruta al archivo config.yaml  [default: config.yaml]

# El framework se selecciona automaticamente segun config:
# model.framework: pytorch  -> trainer_pytorch.py (con AMP)
# model.framework: tensorflow -> trainer_tf.py (Keras)

# Ejemplo:
mlforge train --config examples/01_image_classifier/config.yaml

mlforge export EXPORT

Exporta el modelo entrenado a formatos de despliegue optimizados.

mlforge export --config <path> [opciones]

# Opciones:
  --config, -c     Ruta al config.yaml
  --formats, -f    Formatos: onnx, tflite, coreml, tfjs, edgetpu, all  [default: config]

# Formatos disponibles:
#   onnx     - Universal (web, server, mobile)
#   tflite   - Android, Raspberry Pi (con quantizacion)
#   coreml   - iOS, macOS (Apple Neural Engine)
#   tfjs     - Navegadores web
#   edgetpu  - Google Coral TPU

# Ejemplos:
mlforge export --config config.yaml --formats all
mlforge export --config config.yaml --formats onnx,tflite

mlforge benchmark EXPORT

Ejecuta benchmarks de latencia y throughput comparando diferentes formatos.

mlforge benchmark [opciones]

# Opciones:
  --model-dir, -d   Directorio con modelos exportados
  --config, -c       Config para obtener input_size
  --runs, -n         Numero de iteraciones  [default: 100]

# Ejemplo:
mlforge benchmark --model-dir exported_models/ --config config.yaml --runs 200

mlforge deploy DEPLOY

Genera un proyecto de despliegue completo para la plataforma seleccionada.

mlforge deploy <target> [opciones]

# Targets:
#   android  - Kotlin + CameraX + TFLite
#   ios      - SwiftUI + CoreML + Vision
#   web      - HTML + ONNX Runtime Web
#   edge     - Python + PyCoral/TFLite Runtime
#   list     - Mostrar targets disponibles

# Opciones:
  --model, -m       Ruta al modelo exportado
  --labels, -l      Labels separados por coma
  --input-size      Tamano de entrada  [default: 224]
  --output, -o       Directorio de salida  [default: deploy_output]

# Ejemplos:
mlforge deploy list
mlforge deploy android --model model.tflite --labels "gato,perro,pajaro"
mlforge deploy web --model model.onnx --labels "rosa,tulipan"

mlforge serve DEPLOY

Lanza un servidor de inferencia FastAPI con endpoints REST.

mlforge serve [opciones]

# Opciones:
  --model, -m        Modelo (.onnx o .pth)  [requerido]
  --port, -p         Puerto  [default: 8080]
  --host             Host  [default: 127.0.0.1]
  --labels, -l       Labels separados por coma
  --input-size       Tamano de entrada  [default: 224]
  --architecture, -a Arquitectura (para .pth)
  --num-classes, -n  Numero de clases  [default: 10]

# Endpoints generados:
#   GET  /health        - Estado del servidor
#   GET  /info          - Info del modelo
#   POST /predict       - Clasificar una imagen
#   POST /predict/batch - Clasificar multiples imagenes
#   GET  /docs          - Documentacion Swagger

# Ejemplo:
mlforge serve --model model.onnx --labels "rosa,tulipan" --port 8080

# Llamar al servidor:
curl -X POST http://localhost:8080/predict -F "file=@flower.jpg"

mlforge info UTIL

Muestra las arquitecturas de modelo disponibles para PyTorch y TensorFlow.

mlforge info

# Muestra tabla con:
#   Nombre | Parametros | Tamano | Target recomendado
#   Para ambos frameworks (PyTorch y TensorFlow)

mlforge dashboard UTIL

Lanza el dashboard web con gestion de proyectos, metricas y playground.

mlforge dashboard [opciones]

# Opciones:
  --port, -p   Puerto  [default: 8000]
  --host       Host  [default: 127.0.0.1]

# Funcionalidades del dashboard:
#   - Gestion de proyectos (CRUD)
#   - Lanzar entrenamientos con metricas en tiempo real (WebSocket)
#   - Comparar experimentos
#   - Exportar modelos
#   - Playground de inferencia (drag & drop)
